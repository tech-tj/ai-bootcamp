# AI Engineering Bootcamp Portal

A self-paced learning tracker for the **50-day AI Engineering Bootcamp** — covering LLM APIs, RAG, Agents, Evaluation, and Fine-tuning. Built as a single-file vanilla SPA with no dependencies or build step.

**Live site → [tech-tj.github.io/ai-bootcamp](https://tech-tj.github.io/ai-bootcamp/)**

---

## What it does

- **Dashboard** — split-panel view with progress stats, streak, pace tracking, and a live AI news feed
- **Phase tracker** — 5 phases × 10 days each, mark days complete/skipped with hours logged and notes
- **Adaptive timeline** — end date shifts automatically when days are skipped
- **AI Updates** — dedicated view with 120 articles from 15 RSS feeds, filterable by phase, category, and date
- **AI Frontier Tracker** — personal watchlist for emerging AI topics (watching → learning → building → shipped)
- **Resources** — curated tools, papers, and links organized by phase

## Phases

| Phase | Topic | Duration |
|-------|-------|----------|
| P1 | LLM Foundations & Multi-Provider APIs | 10 days |
| P2 | RAG & Vector Search | 10 days |
| P3 | Agents, MCP & Tool Use | 10 days |
| P4 | Evaluation & Safety | 10 days |
| P5 | Fine-tuning, Serving & MLOps | 10 days |

## Project structure

```
ai-bootcamp/
├── index.html              # Entire app (HTML + CSS + JS, single file)
├── data/
│   ├── plan.js             # Bootcamp curriculum — phases, days, resources
│   ├── state.js            # localStorage state management + OpenModule tracker
│   └── news.json           # Auto-refreshed AI news (generated by GitHub Actions)
└── scripts/
    └── fetch_news.py       # Fetches 15 RSS feeds, tags by phase, writes news.json
```

## News feed

`scripts/fetch_news.py` runs every 6 hours via GitHub Actions. It fetches from:

- **ArXiv** (cs.AI, cs.LG, cs.CL)
- **Lab blogs** — OpenAI, Google AI, HuggingFace, LangChain
- **News** — VentureBeat, TechCrunch, MIT Tech Review, Ars Technica
- **Newsletters** — Import AI, TheSequence
- **Community** — r/MachineLearning, r/LocalLLaMA

Articles are tagged by bootcamp phase relevance and kept for 30 days. The workflow commits updated `news.json` back to `main` automatically.

## Running locally

No build step needed — just serve the directory:

```bash
python3 -m http.server 8099
# open http://localhost:8099
```

To refresh the news feed locally:

```bash
python3 scripts/fetch_news.py
```

## Tech

- Vanilla HTML/CSS/JS — no frameworks, no bundler
- `localStorage` for all user state (progress, notes, frontier topics)
- GitHub Actions for scheduled news refresh
- GitHub Pages for hosting
